# Архитектура (Next.js + Drizzle)

## Стек
- Приложение: Next.js (App Router) + TypeScript (strict)
- Пакетный менеджер: pnpm
- Линтер/форматер: Biome
- БД: PostgreSQL (self-hosted на VPS) + расширение `pgvector` (позже)
- ORM/миграции: Drizzle ORM + drizzle-kit
- Реалтайм: SSE (Server-Sent Events); WebSocket — по необходимости
- LLM: OpenRouter API (модель `meta-llama/llama-4-maverick:free`)

## Слои
- UI (`app/`, `components/`): чат, стейт персонажа, броски кубов
- API (`app/api/*/route.ts`): HTTP и SSE эндпоинты
- Агенты (`lib/agents/*`): GM, Rules, Character, Narrative
- Механики (`lib/mechanics/*`): чистые функции d20, DC, классификация исходов
- Данные (`db/schema.ts`, `db/index.ts`, `drizzle/`): схема БД и миграции
- Конфиг (`env.mjs`): типобезопасная загрузка переменных окружения

## Локализация и сеттинг
- Базовая локализация интерфейса и промптов: RU (русский язык)
- Сеттинг: средневековье; жанр — фэнтези; вдохновлено D&D
- Игрок на старте может указать любой класс персонажа (свободная строка) и краткую информацию о себе

## Поток хода (MVP)
1) Игрок отправляет действие → `/api/chat` (SSE)
2) Параллельно запускаются агенты `Rules` и `Character` (тайм-ауты и отмена)
3) Если требуется проверка: сервер инициирует бросок (или запрашивает его у клиента через `/api/roll`), вычисляет исход детерминированно в `lib/mechanics/*`
4) Агент `Narrative` получает итог проверки и формирует текст
5) Агент `GM` агрегирует результаты, подводит итоги и оформляет ответ
6) Память (MVP): сохраняем сообщения игрока и GM в БД; в следующий ход в контекст кладём последние 12–15 таких сообщений
7) SSE построчно стримит результат на клиент, финализируем ход и пишем записи в БД (в одной транзакции)

Ключевой принцип: все вычислимые вещи (крит/успех, модификаторы, урон и пр.) реализуются как чистые функции на сервере. Модель генерирует только нарратив/оформление.

## API эндпоинты
- `POST /api/chat` — основной чат с SSE стримингом; принимает `{ sessionId?, content, profile? }`
- `POST /api/roll` — бросок d20; принимает `{ sessionId?, skill?, modifiers? }`. Если указаны `sessionId` и `skill`, загружает модификаторы персонажа из БД; иначе использует переданные `modifiers`
- `POST /api/character` — создание/обновление персонажа; принимает `{ className, bio, abilities?, skills? }`
- `GET /api/history` — последние сообщения сессии
- `GET /api/health` — проверка работоспособности сервиса
- `POST /api/session/reset` — сброс текущей сессии

## Решение «бросать/не бросать» (Rules: гибрид)
- Шаг 1 — Heuristic Gate (код): автоуспех/автопровал/нет ставок/успех с ценой
- Шаг 2 — LLM‑классификатор (малый JSON‑ответ): `none | skill | contest | save`, `skill?`, `dc?`, `rationale`, `alternative?`
- Шаг 3 — Sanitizer (код): нормализация навыка и DC к шкале, защита от выходов за границы, фоллбеки
- Итог всегда определяют детерминированные механики; текст — задача Narrative/GM

## Надёжность и наблюдаемость
- Тайм-ауты агентов и `Promise.allSettled` для параллельных запросов
- Circuit breaker: формируем частичный ответ, если один из агентов недоступен
- Трассировка: записываем в таблицы `Turn`/`Message` вход, выход, длительности и версии промптов
- SSE с heartbeat и `retry` для устойчивости

## Аутентификация (MVP)
- Лёгкая аутентификация через httpOnly cookie `rpg_session` (90 дней, SameSite=Lax, Secure в prod)
- Идентификатор используется как `Session.externalId` и позволяет не терять сессию при перезагрузках

## Тестирование
**Фреймворк:** Vitest с поддержкой TypeScript и path alias `@/*`

**Типы тестов:**
- Юнит-тесты чистых механик (d20, DC, критические исходы) в `lib/mechanics/*.test.ts`
- Юнит-тесты эвристических решений Rules Agent в `lib/agents/heuristics.test.ts`
- Контрактные тесты агрегации агентов (фиксированные фейковые ответы) в `lib/agents/*.test.ts`
- Интеграционные тесты `/api/roll` и `/api/chat` (SSE) с фейковым LLM-провайдером в `app/api/**/*.test.ts`

**Конфигурация:**
- Globals включены для удобного синтаксиса (`describe`, `it`, `expect`)
- Node.js окружение
- Исключены: `node_modules/`, `.next/`, `drizzle/`

**Покрытие:**
- `lib/mechanics/dice.test.ts` — классификация d20, применение модификаторов, генерация бросков
- `lib/agents/heuristics.test.ts` — автоматические решения для тривиальных действий и действий со ставками
- `app/api/health/route.test.ts` — проверка работоспособности health endpoint

## Деплой (VPS)
- Node.js LTS, reverse proxy (Nginx/Caddy), системный сервис для Next.js
- `.env`: `DATABASE_URL`, `OPENROUTER_API_KEY`
- Миграции Drizzle запускаются при деплое
