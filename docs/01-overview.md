# Обзор проекта RPGate

## 🎮 Концепция

RPGate - это веб-приложение для текстовых ролевых игр, где в роли Game Master (рассказчика) выступает искусственный интеллект на базе мультиагентной системы.

### Основная идея

Игра вдохновлена механиками Dungeons & Dragons:
- Игрок создает персонажа с уникальными характеристиками
- AI-рассказчик ведет историю и реагирует на действия игрока
- Система бросков кубика (d20) определяет успех/провал действий
- Все приключения сохраняются в базе данных

## 🎯 Ключевые особенности

### 1. Мультиагентная система

Вместо одной большой модели используется **несколько специализированных агентов**:

- **Narrative Agent** - генерирует описания мира и событий
- **Rules Agent** - проверяет правила и определяет сложность проверок
- **Character Agent** - управляет состоянием персонажа
- **GM Agent** (главный) - синтезирует ответы субагентов в финальный результат

**Преимущества:**
- ✅ Разделение ответственности
- ✅ Легкость добавления новых агентов
- ✅ Параллельное выполнение для снижения латентности
- ✅ Возможность использовать разные модели для разных агентов

### 2. Система D&D механик

```
Действие игрока → Проверка Rules Agent → Нужна ли проверка навыка?
                                              ↓
                                         Да / Нет
                                        ↓          ↓
                                   Бросок кубика   Автоуспех
                                        ↓
                                   d20 + модификатор
                                        ↓
                        Критический провал (1) | Провал | Успех | Критический успех (20)
                                        ↓
                                GM формирует результат
```

**Примеры:**

- **Критический провал (1):** "Ты пытаешься перепрыгнуть яму, но спотыкаешься, падаешь и ломаешь ногу"
- **Провал (2-9):** "Ты не смог перепрыгнуть и падаешь в яму"
- **Успех (10-19):** "Ты успешно перепрыгиваешь яму"
- **Критический успех (20):** "Ты элегантно перепрыгиваешь яму, делая сальто в воздухе. Толпа аплодирует!"

### 3. Персистентность данных

- Персонажи хранятся в PostgreSQL
- История приключений с временными метками
- Векторный поиск (pgvector) для долгосрочной памяти
- AI может "вспоминать" прошлые события

## Локализация
- По умолчанию проект направлен на русскоговорящий сегмент RU
- В будущем планируется мультиязычность

## ⚔️ Сеттинг и персонаж
- Сеттинг: средневековье; жанр — фэнтези; вдохновлено D&D
- Игрок может указать любой класс персонажа (свободная строка)
- Игрок может добавить краткую информацию о себе/персонаже (био)

## 🧠 Память в MVP
- На первом этапе (MVP) используется контекстная память из последних 12–15 сообщений, идущих между Игроком и GM
- Сообщения от вспомогательных агентов (Rules/Character/Narrative) в контекст не включаются
- Векторная память (pgvector) планируется на следующих этапах

## 🎨 Целевая аудитория

- Любители текстовых RPG
- Фанаты D&D и настольных игр
- Игроки, которым интересен AI-driven сторителлинг
- Те, кто хочет играть в D&D соло без DM

## 🌟 Уникальность проекта

### Отличия от существующих решений:

1. **Мультиагентная архитектура** - более консистентный и качественный геймплей
2. **Модульность** - легко добавлять новые механики и системы
3. **Векторная память** - AI помнит важные события на протяжении всей игры
4. **Open source** - возможность модификации и расширения

## 📈 Видение развития

### Краткосрочная перспектива (3-6 месяцев)
- Качественный MVP с базовыми механиками
- Система боя
- Инвентарь и предметы
- Векторная память

### Среднесрочная перспектива (6-12 месяцев)
- Система магии
- Крафтинг
- Квесты и достижения

### Долгосрочная перспектива (12+ месяцев)
- Компаньоны и NPC
- Мультиплеер (совместные приключения)
- Создание собственных миров
- Marketplace для пользовательского контента

## 🎪 Масштабируемость

### Под масштабируемостью понимается:

**✅ Наращивание функционала:**
- Добавление новых агентов
- Новые игровые механики
- Расширение игрового мира
- Интеграция дополнительных AI моделей

**❌ НЕ горизонтальное масштабирование нагрузки:**
- В MVP не используется Redis
- Простая архитектура для быстрой разработки

*(Горизонтальное масштабирование можно добавить позже при необходимости)*

## 📝 Принципы разработки

1. **Модульность** - каждая система независима
2. **Расширяемость** - легко добавлять новое
3. **Тестируемость** - автоматические тесты для всего
4. **Простота** - не переусложняем MVP
5. **Документированность** - код понятен другим разработчикам
