# Анализ контекста памяти в RPGate

## Обзор

В RPGate используются **два типа памяти** для контекста:

1. **Быстрая память (Short-term)** - последние сообщения в истории чата
2. **Долгосрочная память (Long-term)** - Personal Memory система с векторным поиском

## Быстрая память (History)

### Размер контекста

**Текущее значение:** `15 сообщений`

**Код:** `app/api/chat/route.ts`
```typescript
// Build last 15 messages (player/gm) history for context
const historyRows = await db
  .select({ role: messages.role, content: messages.content })
  .from(messages)
  .where(
    and(
      eq(messages.sessionId, session.id),
      inArray(messages.role, ["player", "gm"]),
    ),
  )
  .orderBy(desc(messages.createdAt))
  .limit(15);  // ← Здесь задаётся размер
```

### Что включается

- ✅ Сообщения игрока (`role: "player"`)
- ✅ Сообщения GM (`role: "gm"`)
- ❌ Системные сообщения (`role: "system"`) - НЕ включаются

### Формат в промпте

**Сжатый формат** для экономии токенов:

```
История:
P: Я иду в таверну
G: Ты входишь в таверну Золотой Дракон
P: Я заказываю эль
G: Бармен наливает тебе кружку эля
...
```

**Код:** `lib/agents/narrative-llm.ts`
```typescript
const hist = history
  .map((m) => `${m.role[0].toUpperCase()}: ${m.content}`)
  .join("\n");
```

### Использование

History передаётся в:

1. **Heuristic Gate** - для определения unknown entities
   ```typescript
   const heuristicResult = analyzeMemoryNeed(input.content, history);
   ```

2. **Narrative Agent** - для генерации контекстуального ответа
   ```typescript
   streamNarrative(input, rules, outcome, history, characterProfile, memories);
   ```

## Долгосрочная память (Personal Memory)

### Размер контекста

**Текущее значение:** `5 воспоминаний` (top-K)

**Код:** `lib/memory/config.ts`
```typescript
export const RETRIEVAL_CONFIG = {
  DEFAULT_LIMIT: 5,  // ← Top-K результатов
  DEFAULT_SIMILARITY_THRESHOLD: 0.7,
  DEFAULT_TIMEOUT_MS: 2000,
};
```

### Что включается

- ✅ Важные события (location, npc, item, event, decision)
- ✅ Только релевантные (similarity > 0.7)
- ✅ Отсортированные по similarity (DESC)

### Формат в промпте

**Структурированный формат:**

```
Воспоминания:
1. [Ход 3] Ты входишь в город Новгород. Улицы полны людей...
2. [Ход 5] Торговец Иван приветствует тебя. Он предлагает...
3. [Ход 8] Ты находишь древний меч с рунами...
```

**Код:** `lib/agents/narrative-llm.ts`
```typescript
let memoriesText = "";
if (memories && memories.length > 0) {
  memoriesText = `\n\nВоспоминания:\n${memories
    .map((m, i) => `${i + 1}. [Ход ${m.turnNumber}] ${m.summary}`)
    .join("\n")}`;
}
```

### Использование

Memories передаются только в **Narrative Agent** для генерации ответа.

## Сравнение

| Параметр | Быстрая память | Долгосрочная память |
|----------|----------------|---------------------|
| **Размер** | 15 сообщений | 5 воспоминаний |
| **Охват** | Последние ~7-8 ходов | Вся история сессии |
| **Триггер** | Всегда | 20-30% запросов (heuristic) |
| **Поиск** | Хронологический | Семантический (векторный) |
| **Latency** | ~0ms (из БД) | ~700ms (embedding + search) |
| **Формат** | Полные сообщения | Краткие summary |
| **Использование** | Heuristic + Narrative | Только Narrative |

## Оценка токенов

### Быстрая память (15 сообщений)

**Средний размер:**
- 1 сообщение игрока: ~20 токенов
- 1 сообщение GM: ~50 токенов
- Итого за ход: ~70 токенов
- **15 сообщений ≈ 525 токенов**

### Долгосрочная память (5 воспоминаний)

**Средний размер:**
- 1 summary: ~40 токенов
- Форматирование: ~10 токенов
- **5 воспоминаний ≈ 210 токенов**

### Итого контекст

**Без долгосрочной памяти:**
- История: 525 токенов
- Персонаж: ~50 токенов
- Действие: ~20 токенов
- **Итого: ~595 токенов**

**С долгосрочной памятью:**
- История: 525 токенов
- Воспоминания: 210 токенов
- Персонаж: ~50 токенов
- Действие: ~20 токенов
- **Итого: ~805 токенов**

**Увеличение:** +35% токенов при использовании памяти

## Рекомендации

### Текущие настройки оптимальны

✅ **15 сообщений** - хороший баланс между контекстом и токенами
- Покрывает ~7-8 последних ходов
- Достаточно для понимания текущей ситуации
- Не перегружает промпт

✅ **5 воспоминаний** - достаточно для контекста
- Top-5 самых релевантных событий
- Не перегружает промпт
- Даёт достаточно информации для GM

### Возможные оптимизации

#### 1. Динамический размер history

Вместо фиксированных 15 сообщений, использовать лимит по токенам:

```typescript
// Вместо .limit(15)
const MAX_HISTORY_TOKENS = 500;
let historyRows = [];
let totalTokens = 0;

for (const msg of allMessages) {
  const tokens = estimateTokens(msg.content);
  if (totalTokens + tokens > MAX_HISTORY_TOKENS) break;
  historyRows.push(msg);
  totalTokens += tokens;
}
```

**Плюсы:**
- Более точный контроль над размером промпта
- Адаптируется к длине сообщений

**Минусы:**
- Сложнее реализация
- Нужна функция estimateTokens()

#### 2. Сжатие старых сообщений

Сжимать сообщения старше N ходов:

```typescript
const history = historyRows.map((m, i) => {
  if (i < 5) {
    // Последние 5 - полные
    return m;
  } else {
    // Старые - сжатые
    return {
      ...m,
      content: m.content.slice(0, 50) + "..."
    };
  }
});
```

**Плюсы:**
- Экономия токенов
- Сохранение общего контекста

**Минусы:**
- Потеря деталей
- Может сломать понимание контекста

#### 3. Адаптивный limit для memories

Использовать больше воспоминаний для сложных запросов:

```typescript
const limit = heuristicResult.confidence > 0.9 
  ? 10  // Высокая уверенность - больше контекста
  : 5;  // Обычный запрос

const memories = await retrieveMemories(sessionId, query, { limit });
```

**Плюсы:**
- Больше контекста для важных запросов
- Экономия на простых запросах

**Минусы:**
- Непредсказуемый размер промпта
- Может превысить лимит токенов

## Мониторинг

### Метрики для отслеживания

1. **Средний размер промпта** (токены)
   - Без памяти: ~600 токенов
   - С памятью: ~800 токенов

2. **Hit rate долгосрочной памяти**
   - Целевой: 20-30% запросов
   - Текущий: проверить через логи

3. **Средний count воспоминаний**
   - При retrieval: 2-3 воспоминания
   - Максимум: 5 воспоминаний

4. **Latency**
   - Без памяти: ~2-3 секунды
   - С памятью: ~3-4 секунды (+700ms)

### Логирование

Добавить метрики в логи:

```typescript
console.log('[Context]', {

  historyTokens: estimateTokens(hist),
  memoriesCount: memories?.length || 0,
  memoriesTokens: memories ? estimateTokens(memoriesText) : 0,
  totalTokens: totalPromptTokens

```
