# Тестирование UI индикаторов Personal Memory

## Быстрый старт

1. **Запустить dev сервер** (уже запущен):
   ```bash
   pnpm dev
   ```

2. **Открыть браузер**: http://localhost:3000

3. **Создать персонажа** (опционально):
   - Класс: "Воин"
   - Биография: "Опытный путешественник"

## Тестовые сценарии

### Сценарий 1: Возврат в знакомое место ✨

**Цель:** Проверить полный flow: searching → found → stored

**Шаги:**
1. Отправить: `"Я прибываю в город Новгород"`
   - Ожидание: GM ответит о прибытии
   - Ожидание: Через 1.5 сек появится 🟢 "Момент сохранён в памяти"

2. Отправить: `"Я встречаю торговца Ивана"`
   - Ожидание: GM ответит о встрече
   - Ожидание: Через 1.5 сек появится 🟢 "Момент сохранён в памяти"

3. Отправить: `"Я возвращаюсь в Новгород"`
   - Ожидание: 🔵 "Поиск в памяти..." (пульсирующий)
   - Ожидание: 🟣 "Найдено N воспоминаний"
   - Ожидание: GM ответит с учётом воспоминаний
   - Ожидание: Через 2 сек появится 🟢 "Момент сохранён в памяти"

**Что проверяем:**
- ✅ Индикатор "searching" появляется сразу
- ✅ Индикатор обновляется на "found" с правильным count
- ✅ Индикатор "stored" появляется после задержки
- ✅ GM использует воспоминания в ответе

## Troubleshooting

### Проблема: Индикаторы не появляются

**Решение:**
1. Проверить консоль браузера на ошибки
2. Проверить что dev сервер запущен
3. Проверить Network → EventStream на наличие событий
4. Перезагрузить страницу (Cmd+R / Ctrl+R)

### Проблема: "Поиск в памяти..." не обновляется на "Найдено"

**Причины:**
- Retrieval не нашёл воспоминаний (similarity < 0.7)
- Timeout retrieval (> 2 секунды)
- Ошибка в embedding API

**Решение:**
1. Проверить логи в консоли сервера
2. Проверить что AITUNNEL_API_KEY настроен
3. Попробовать более конкретный запрос

### Проблема: Индикатор "stored" не появляется

**Причины:**
- Таймаут не сработал (проверить setTimeout)
- Компонент размонтировался до показа

**Решение:**
1. Проверить что прошло 1.5-2 секунды после ответа
2. Проверить консоль на ошибки React
3. Попробовать отправить новое сообщение

### Проблема: Дублирование индикаторов

**Причины:**
- Быстрая отправка нескольких сообщений
- Таймауты не отменяются

**Решение:**
- Это известное ограничение
- Подождать завершения предыдущего хода
- Или игнорировать (не критично)

## Метрики для проверки

### Производительность

- **Heuristic:** < 1ms (мгновенно)
- **Retrieval:** 500-1000ms (видимая задержка)
- **UI update:** < 16ms (60 FPS)

### UX

- **Индикатор появляется:** < 100ms после события
- **Анимация плавная:** 60 FPS
- **Текст читаемый:** Контраст > 4.5:1 (WCAG AA)
